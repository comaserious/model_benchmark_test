## 4. 모델별 상세 분석

### 🏆 Gemma-4-26B-A4B-it (A100) — 종합 가성비 1위

**왜 강력한가?**
- MoE 구조: 26B total / 4B active → A100 80GB에서 여유 있게 구동
- 단일 사용자 TPS **103.9** (EXAONE-AWQ 65.6의 1.6배)
- 50 동시 사용자에서도 avg TTFT=1.14s, avg E2E=21.2s → SLA 완벽 충족
- 품질: MMLU-Pro ~82, GPQA 82.3 (전체 오픈소스 3위권)

**주의사항**
- 10 동시 사용자 구간에서 TTFT 이중 분포 관찰 (일부 사용자 8.8s). 20u 이후 다시 안정됨
- 이 현상은 MoE 배치 정렬 오버헤드로 추정됨

```
@50u: TTFT=1.14s | TPS=48.7 | E2E=21.2s ✅
```

### ⭐ EXAONE-4.0-32B-AWQ (A100) — 한국어 서비스 1순위

**왜 강력한가?**
- TTFT 저하율이 1.1×로 **모든 모델 중 가장 안정적** (1u~50u 거의 변동 없음)
- AWQ 양자화: FP8 대비 TPS가 65.6 vs 41.6으로 **57% 우수**
- MMLU-Pro 81.8, AIME 2025 **85.3** (추론 능력 최상위급)
- KMMLU-Pro, KMMLU-Redux 포함 한국어 공식 평가 수행

**FP8 대신 AWQ를 추천하는 이유**
| 지표 | AWQ | FP8 |
|------|-----|-----|
| @1u TPS | **65.6** | 41.6 |
| @50u TPS | **40.3** | 30.0 |
| @50u E2E | **21.2s** | 23.4s |
| TTFT 저하율 | **1.1×** | 2.2× |

```
@50u: TTFT=1.87s | TPS=40.3 | E2E=21.2s ✅
```

### 🚀 gpt-oss-20b (A100) — MoE 처리량 최강

**왜 강력한가?**
- 단일 사용자 TPS **213.6** — 테스트된 모든 모델 중 압도적 1위
- 50 동시 사용자에서도 TPS=113.8 유지 (Qwen3-8B의 1.6배)
- MoE 20B total / 3.6B active → VRAM 효율 우수
- o3-mini 수준 품질

**제한사항**
- @1u TTFT=2.99s로 이미 한계치 (5u부터 TTFT 초과)
- 고품질 응답 생성 시 E2E가 길어지는 경향 (avg 19.6s @50u)
- 처리량 중심 서비스 (API 배치, 비동기 처리)에 최적

```
@50u: TTFT=4.89s ⚠️ | TPS=113.8 ★ | E2E=19.6s ✅
(TTFT SLA 완화 시 50+ 동시 사용자 가능)
```

### ⚡ Qwen3-8B (A100/RTX5090) — 경량 서비스 최강

**왜 강력한가?**
- A100 기준 모든 부하 구간 TTFT < 1.2s — **가장 응답성 높은 모델**
- RTX5090×2 구성 시 단일 사용자 TPS=127.6 (A100 1GPU의 1.6배)
- Thinking mode 지원 (enable_thinking 옵션으로 추론 능력 강화 가능)
- 저비용 GPU(RTX 4090, RTX 5090)에서도 높은 성능

**RTX5090 ×2 vs A100 ×1 비교 (Qwen3-8B)**
| 구간 | RTX5090×2 avg TPS | A100×1 avg TPS |
|------|-----------------|--------------|
| @1u | 127.6 | 79.9 |
| @10u | 99.4 | 76.4 |
| @50u | 86.4 | 70.9 |

```
@50u (A100): TTFT=0.89s | TPS=70.9 | E2E=17.5s ✅
@50u (RTX5090×2): TTFT=0.86s | TPS=86.4 | E2E=15.0s ✅
```

### 🔬 Gemma-4-31B-it (A100) — 최고 품질, 전문 업무용

**왜 주목해야 하는가?**
- 전체 오픈소스 중 MMLU-Pro 85.2, GPQA 84.3, LiveCodeBench 80.0 — 최상위권
- TTFT 안정성 최고 수준 (50u에서도 1.9s, 저하율 1.1×)
- 응답 자체의 정보 밀도와 정확도가 최고

**제한사항 — 실서비스 주의**
- **TPS=17~24로 모든 모델 중 최저** (대용량 응답 생성 오버헤드)
- avg E2E @50u = **53.7s** (SLA 위반)
- 실시간 채팅 서비스보다 **비동기 문서 분석, 고품질 보고서 생성**에 적합

```
@50u: TTFT=1.90s ✅ | TPS=17.9 | E2E=53.7s ⚠️
→ 비동기/배치 처리 권장
```
